iOS 学习笔记—@synthesize合成指令

發表於 2015-07-17

许多编程语言中，在程序员编写一个

@property NSString *firstName;

@property (nonatomic,strong) NSString *someProperty;//声明一个变量

@sythesize someProperty = _someProperty； // 必须使用 合成指令告诉编译器，其才会生成该属性需要的成员变量。

@interface parentViewController（）
@property (nonatomic,strong) NSString *someProperty; //父类的属性
@end

@implementation childViewController
@sythesize someProperty = _someProperty；
@(void)someMethod
{
self.someProperty //循环引用..
}
@end
```

这么一来简化了程序员的工作量，但是当成员变量出现在父类的情况下，子类如果需要调用父类的成员变量时，编译器不会给子类也生成相应的下划线成员变量。Xcode 则推荐我们使用self关键字。但是通过self.someProperty调用父类的成员变量时候，是调用属性的getter方法，getter 调用时候再 getter 自己的时候就会形成循环引用的问题。

明白了这个之后，往后在子类调用父类的变量时候，手动调用一下@sythesize就可以了。
